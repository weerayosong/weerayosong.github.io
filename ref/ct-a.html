2. RESTful API Routes

Method Endpoint Payload (Body) Description
GET /api/v1/patients None Retrieve all active patients in the triage queue.
POST /api/v1/patients { name, symptoms, urgency } Register a new patient. Status defaults to WAITING.
PATCH /api/v1/patients/{id}/status { status } Update specific patient status (WAITING, TREATMENT, DISCHARGED).

3. Backend Architecture (Spring Boot)

Controllers (Presentation)

  • PatientController.java: Handles incoming HTTP requests, extracts variables, and maps to the Service layer. Cross-Origin (CORS) is configured here.

Services (Business Logic)

  • PatientService.java: Contains logic for state transitions.

Repositories (Data Access)

  • PatientRepository.java: Extends JpaRepository to handle CRUD operations without boilerplate SQL.

Entities (Domain Model)

  • Patient.java: Maps to PostgreSQL. Fields include ID, Name, Symptoms, UrgencyLevel, Status, and CreatedAt.

4. Frontend Architecture (Angular)

Components (UI & Logic)

  • TriageBoardComponent: The main smart component that holds the state of the queue.
  • RegistrationFormComponent: Handles user input and triggers validation.

Services (API & State)

  • TriageApiService.ts: Utilizes Angular's HttpClient to send requests to the backend.
